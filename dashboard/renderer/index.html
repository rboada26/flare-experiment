ABOUT

PARTNER-LAB captures network traffic from 6 federated learning (FL) architectures and trains a Random Forest to fingerprint each one from packet metadata alone.

CLASSES & ARCHITECTURES
LSTM (r)
Long Short-Term Memory
Gated memory cells capture long-range temporal dependencies. FL use: health monitoring, sensor fusion.
GRU (r)
Gated Recurrent Unit
Efficient LSTM variant — fewer parameters, no separate output gate. FL use: edge NLP, lightweight sequences.
BiLSTM (r)
Bidirectional LSTM
Processes sequences in both directions for richer context. FL use: medical time-series, speech recognition.
SimpleCNN (c)
3-Layer Convolutional Network
Lightweight baseline CNN for image classification. FL use: image triage, quality inspection.
MobileNet (c)
MobileNetV2
Depthwise-separable convolutions for mobile/edge deployment. FL use: on-device vision, mobile inference.
ResNet-18 (c)
Residual Network
Skip connections enable deep training without vanishing gradients. FL use: medical imaging, robust visual recognition.
TRAINING DATASET

CIFAR-10 — 60,000 colour images (32×32) across 10 classes: plane, car, bird, cat, deer, dog, frog, horse, ship, truck.

Using a neutral dataset ensures fingerprints reflect architecture differences, not dataset complexity.

⚠ WHY THIS IS A VULNERABILITY

Even with TLS encryption, FL coordination traffic carries measurable side-channel information:

Architecture Disclosure Traffic timing and volume reveal whether an org trains CNNs or RNNs — exposing proprietary R&D direction.
Gradient Leakage Knowing the architecture is a prerequisite for model-inversion and gradient inference attacks.
Targeted Poisoning Architecture-specific backdoors become viable once an adversary identifies the model type from network traffic alone.
Competitive Intel In healthcare and finance, knowing a competitor's model family leaks strategic intent.

FLARE achieves 97.7% F1 using only packet-size windows — motivating traffic obfuscation, timing jitter, and dummy-packet injection as defenses.
